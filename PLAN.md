# 타로 노트

기술 이름(slug): `tarot-note`

## 한 줄 소개
매일 뽑은 타로 카드를 이미지와 함께 기록하고 되돌아보는 나만의 디지털 타로 일지

## 핵심 흐름
1. 78장 중 카드를 고른다 — 날짜·질문·해석을 적어 저장
2. 기록 목록에서 카드 이미지로 과거 기록을 한눈에 본다
3. 상세 화면에서 그 당시 해석을 다시 읽고, 새로운 코멘트(리뷰)를 덧붙인다

## 참고 앱/사이트
없음

- 로그인: 필요 없음 (나 혼자 쓰는 도구)
- LLM API: 불필요

## 디자인
- 강조색: `#004ac6` (Stitch primary) — 버튼·링크·강조. 보라색 그라데이션은 쓰지 않는다.
- 배경: `#faf8ff` · 글자: `#131b2e` · 카드: `#ffffff` · 테두리: `#c3c6d7`
- 보조 표면: surface-container `#eaedff` · surface-container-low `#f2f3ff`
- 상태색: 성공 `#16A34A` · 오류 `#ba1a1a`
- 폰트: Noto Sans KR
- 모서리(radius): 12px (`rounded-xl`) · 그림자: 옅게 카드에만 · 간격: 넉넉하게
- 레이아웃 원칙: 기록 목록·달력·도감은 그리드, 입력·상세는 중앙 1단
- 네비게이션: 상단 탭바 — 기록하기·모아보기·달력·도감 (통계·설정은 다음에)
- 카드 이름 표기: "영어이름 / 한글이름" 병기 (예: "The Magician / 마법사")

## 화면
1. **기록 목록** ("나의 성찰 기록") — `/`
   - 보임: 카드 썸네일+날짜+카드이름 그리드(1순위) → 새 기록 버튼(우상단)
   - 동작: 기록 카드 클릭 → 상세 이동 · '새 기록' 버튼 → 입력 화면
   - 데이터: 읽음: readings
   - 상태: 로딩 스피너 · 0건이면 "첫 기록을 남겨볼까요?" 안내 · 오류 메시지

2. **새 기록** ("오늘의 기록") — `/new`
   - 보임: 카드 선택 그리드(1순위, 78장 이름 목록) → 날짜·질문·해석 입력폼
   - 동작: 카드 선택 → 텍스트 입력 → 저장
   - 데이터: 씀: readings
   - 상태: 저장 중 비활성화 · 저장 완료 → 목록 이동 · 필수항목 누락 오류

3. **기록 상세** — `/readings/[id]`
   - 보임: 카드 이미지(크게, 1순위) → 날짜·질문·해석 · 리뷰 목록 · 리뷰 입력창
   - 동작: 리뷰 텍스트 작성 후 추가 · 기록 삭제
   - 데이터: 읽음: readings, reviews · 씀: reviews
   - 상태: 로딩 · 리뷰 없을 때 "지금 되돌아보고 코멘트를 남겨보세요" 안내

4. **달력** — `/calendar`
   - 보임: 월별 달력(1순위) — 기록 있는 날에 카드 미니 이미지 표시
   - 동작: 날짜 클릭 → 해당 날의 기록 상세 이동 · 월 앞뒤 이동
   - 데이터: 읽음: readings (날짜 필드 기준)
   - 상태: 로딩 · 기록 없는 월은 달력만 표시

5. **카드 도감** — `/cards`
   - 보임: 78장 카드 그리드(1순위) — 기록한 적 있는 카드는 강조 표시
   - 동작: 카드 클릭 → 그 카드로 남긴 기록 목록 (상세 이동)
   - 데이터: 읽음: readings (card_name 기준 필터)
   - 상태: 로딩 · 기록 없는 카드는 흐리게

## 데이터 (Supabase 테이블)
- **readings**
  - `id` uuid — 기본 키
  - `card_name` text — 카드 이름 (78장 중 하나)
  - `card_image_url` text — 카드 이미지 URL
  - `reading_date` date — 카드를 뽑은 날짜
  - `question` text — 그날의 질문 또는 상황
  - `interpretation` text — 해석 / 메모
  - `created_at` timestamptz
  - 관계: 없음

- **reviews**
  - `id` uuid — 기본 키
  - `reading_id` uuid — readings.id 참조
  - `review_text` text — 리뷰 내용
  - `reviewed_at` timestamptz
  - 관계: reading_id → readings.id
