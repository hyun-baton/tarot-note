## 메뉴 개편 + 카드 이미지 교체 (2026-05-31)
**무엇을:** 도감·달력 페이지 삭제, 달력을 모아보기(`/`)에 통합(기록 있는 날 카드 썸네일 표시·오늘 강조). 새 기록은 카드 이미지 그리드 → 이름 검색란으로. 카드 이미지를 Smith-Waite 보더리스(yunruse/tarot color1910)로 통일.
**왜 이렇게:** 화면 수를 줄여 단순화(나 혼자 쓰는 도구). 검색이 78장 그리드보다 빠름. 보더리스가 사용자 실물 카드와 가까움.
**핵심 결정:** 기존 DB의 옛 이미지 URL 대신 `CARD_BY_NAME[card_name]`로 최신 이미지를 조회(기록 이름→이미지 매핑). 흰 테두리는 `scale-[1.08]`로 잘라 보더리스 표현. 사진 업로드→AI 인식(방식 2)은 LLM API 불필요 원칙·범위 고려해 보류, 검색(방식 1) 유지로 결정.
**막힌 점:** Tailwind v4에서 `grid-cols-[200px,1fr]` 파싱 깨짐 → flex로 해결. next.config에 media.githubusercontent.com 이미지 도메인 추가 필요.

## 스프레드(다중 카드 배치) (2026-05-31)
**무엇을:** 기록 1건 = 카드 여러 장(1~15) + 각 카드의 캔버스 위치(x/y %). 새 기록에 스프레드 편집기 추가 — 템플릿(데일리·쓰리·십자·켈틱·자유) 선택 + 카드 검색 추가 + 드래그로 위치 조정. 단일 카드는 1장짜리 스프레드로 통합. 상세는 2장↑이면 전체폭 캔버스, 1장이면 기존 좌우 2단.
**데이터:** readings에 `cards jsonb`(`[{name,nameKo,imageUrl,x,y}]`) + `spread_name text` 추가 (supabase/003_spread_cards.sql, --linked 적용). 표지=cards[0], 달력 썸네일은 card_name→CARD_BY_NAME으로 이미지 해석.
**핵심 결정:** 자리별 개별 해석은 안 함(전체 해석 하나) — 사용자 결정. 배치는 템플릿+드래그 둘 다. 드래그는 Pointer Events(마우스+터치), 위치는 백분율로 저장해 반응형 유지.
**막힌 점/주의:** 검증 중 기존 샘플 기록(펜타클 10, 5/30)이 DB에서 사라져 복원함(restore SQL). 코드엔 타 기록 삭제 경로 없음(새 기록=INSERT만). 향후 새 기록 저장이 다른 기록을 지우지 않는지 한 번 더 관찰 권장.
